# Scratch renewal reference

## Sol's deletion pipeline

Sol's `/scratch` uses a layered deletion policy (see the official docs:
<https://docs.rc.asu.edu/scratch#scratch-limitations>):

| Stage          | Trigger                        | Sol action                      |
|----------------|--------------------------------|---------------------------------|
| inactive       | 45+ days without access        | weekly warning emails begin     |
| over 90 days   | 90+ days without access        | subject to deletion             |
| pending removal| final window (~<7 days left)   | "last chance" notice            |
| removed        | deleted                        | gone                            |

Each stage corresponds to a CSV Sol writes in the user's `$HOME`:

| CSV file                             | Stage key    |
|--------------------------------------|--------------|
| `scratch-dirs-inactive.csv`          | `inactive`   |
| `scratch-dirs-over-90days.csv`       | `over90`     |
| `scratch-dirs-pending-removal.csv`   | `pending`    |
| `scratch-dirs-removed.csv`           | (post-facto) |

All CSVs share a `Directory` column listing the flagged leaf directories.
Other columns vary (`Last Used`, `Days Since Last Used`, `File Count`,
`Size (GiB)`, ...) — `sol_renew.py` only reads `Directory`.

These CSVs are regenerated by Sol on its own cadence. A dry-run may show
directories that have already been touched in a previous run but not yet
removed from the CSV — expected.

## `.solignore` syntax

Lives at `$HOME/.solignore`. Gitignore-style patterns, but the semantics
are **inverted**: a matched path is *kept* (touched). Nothing outside a
match is ever touched.

| Pattern                               | Meaning                                             |
|---------------------------------------|-----------------------------------------------------|
| `# comment`                           | ignored                                             |
| `/scratch/alice/project`             | bare path = directory prefix, matches path/**       |
| `/scratch/alice/logs/*.log`          | glob; `*` does not cross `/`                        |
| `/scratch/alice/data/**`             | `**` matches any depth                              |
| `!/scratch/alice/data/tmp/**`        | `!` negates — excludes this subtree                 |
| `/scratch/alice/cache/`              | trailing `/` matches only if it's a directory       |

Rules are evaluated top-to-bottom; the last matching rule wins
(same as gitignore). Unmatched paths are **skipped** by default.

## `sol_renew.py` invocations

```shell
# default: protect-list + all non-removed stages
sol_renew.py

# preview only
sol_renew.py --dry-run -v

# only attack the most urgent bucket
sol_renew.py --stage pending

# custom locations
sol_renew.py --csv-dir /tmp/sol-csvs --solignore /tmp/keep.ignore

# crank up parallelism for a slow NFS night
sol_renew.py -j 16
```

Exit codes:

- `0` — all flagged + kept directories touched successfully
- `1` — at least one directory failed
- `2` — no rules loaded (empty or missing `.solignore`) — nothing done

## Performance notes

- Each CSV row is already a leaf directory in Sol's scan. `sol_renew.py`
  runs exactly one `find | xargs touch` pipeline per row. No wholesale
  walk of `/scratch`.
- `touch -a -m -c` updates both atime and mtime. Sol's "Last Used"
  classifier is access-time based; atime is what buys the renewal.
- NFS `touch` on a dir with tens of thousands of small files can sit for
  minutes. The script prints one progress line per directory when it
  finishes — an agent watching the output should not interpret a stalled
  counter as a hang.

## Emergency single-path touch

If you need to refresh a single path outside the CSV workflow:

```shell
find /scratch/$USER/my_dir -type f -print0 \
  | xargs -0 -r -n 500 touch -a -m -c --
```

That is the same primitive `sol_renew.py` uses internally.
